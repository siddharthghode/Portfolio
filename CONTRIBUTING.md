# Contributing to Siddharth’s Portfolio

Thank you for your interest in contributing! This repository is mostly a personal portfolio site, but contributions are welcome for code quality, accessibility, and documentation.

## Issues

- Open a new issue for bug reports, content improvements, or feature ideas.
- Use clear titles and include steps to reproduce if applicable.

## Pull Requests

1. Fork the repository.
2. Create a feature branch: `git checkout -b feature/your-idea`.
3. Install dependencies: `npm install`.
4. Run lint and formatting:
   - `npm run lint`
5. Make your changes.
6. Commit with clear, descriptive messages.
7. Push to your fork and open a PR with a short summary and testing notes.

## Local testing

- `npm run dev` for local development.
- `npm run build` to validate production build.

## Coding style

- React components in `src/`
- Tailwind utility classes in CSS files
- Avoid committing secrets (EmailJS keys should be stored in environment variables)
